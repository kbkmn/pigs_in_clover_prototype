# pigs_in_clover_prototype

Лабиринт с шариком для смарт-часов. Этот файл — **источник правды по механикам и картинке**. По нему прототип можно собрать заново на чём угодно.

## Правило для Claude (и любого, кто здесь работает)

1. Любое изменение геймплея или света сначала описывается здесь, потом пишется в коде. Код, расходящийся с этим файлом, — баг кода.
2. Числа в этом файле — нормативные. Меняешь баланс — меняешь таблицу здесь, в том же коммите.
3. Ввод навсегда: **тап** и **колесо** (корона). Механика, требующая третьего ввода, не подходит.
4. Прототип браузерный, но целится в смарт-часы: один аналоговый канал (об/с) + дискретный тап.

## Что это

Классическая детская игрушка: деревянный лабиринт, металлический шарик, дырка-выход. Наклона нет — вместо него **вращение лабиринта вокруг шарика** колесом. Один уровень в виде буквы П. Шарик стартует внизу левой ноги, выход — дырка в полу внизу правой.

## Относительность (главный принцип)

Игрок крутит **камеру и гравитацию**, а лабиринт и его свет стоят на месте:

- на экране шарик всегда в центре, лабиринт вращается вокруг него;
- гравитация всегда вниз по экрану;
- свет уровня прибит к лабиринту и вращается вместе с ним (запечён в текстуру);
- шейдинг шарика — экранный (свет как бы от зрителя), но **отражение** уровня в хроме вращается вместе с лабиринтом.

## Ввод

- **Колесо** — единственный непрерывный канал. Единица — обороты в секунду (об/с), реальный человек выдаёт 0…2.5.
- 1 об/с колеса = **2.4 рад/с** вращения лабиринта (~137°/с). Вращение всегда вокруг текущей позиции шарика.
- **Тап** — рестарт после финиша. Во время игры не используется.
- **R** — полный сброс в любой момент (служебное).
- В браузере: скролл мыши / ↑↓ / драг короны; сглаживание rps с decay 7.5; отпускание драга ловится на `window` + `lostpointercapture` + `blur`.

## Физика

Симуляция в **локальной системе лабиринта**: шарик хранится в координатах уровня, кручение колеса поворачивает вектор гравитации; вектор скорости шарика доворачивается на −dθ, чтобы мировая инерция сохранялась. Ощущение: лабиринт крутится, шарик по инерции продолжает лететь как летел.

| параметр | значение |
| --- | --- |
| гравитация | 1500 px/с² |
| отскок от стен | 0.3 |
| трение о стены | 1.1/с (только тангенциальное, в контакте) |
| сопротивление воздуха | 0.04/с |
| потолок скорости | 1600 px/с |
| сабстепов на кадр | 4 |

Коллизии: круг против 8 отрезков контура, ближайшая точка + выталкивание по нормали.

**Инерция неприкосновенна.** Трение о стены выше ~2/с душит качение (шарик постоянно в контакте с полом) и убивает игру. Проверка: при постоянном наклоне скорость качения должна монотонно расти.

## Уровень

Буква П в локальных координатах, px:

| параметр | значение |
| --- | --- |
| габариты | 512 × 620 |
| толщина ног | 112 |
| толщина перекладины | 68 |
| толщина стенок (визуальная) | 24 |
| радиус шарика | 15 |
| радиус дырки | 24 |
| старт | центр левой ноги, у самого низа |
| дырка | центр правой ноги, y = низ − 48 |

Победа: расстояние центра шарика до центра дырки < 0.62 × радиуса дырки → фаза `fall` (0.45 с, шарик всасывается и сжимается) → `won` («ФИНИШ», тап — заново).

## Экран и UI

- Прототип рисует часы: безель, скруглённый экран 396×484, корона сбоку (драг + скролл), измеритель об/с.
- **Стрелка на выход**: жёлтая, вокруг шарика на радиусе 52, в экранных координатах. Гаснет при приближении к дырке (fade 90…180 px).
- **Миникарта**: сверху по центру, ширина 44 px, **не вращается**. Контур П, белая точка — шарик, жёлтая — дырка. Подложка — слабый радиальный градиент из полупрозрачного чёрного (0.38) в прозрачный.
- Текст поверх геймплея — с мягкой тенью без сдвига, blur 32.
- Пауза при скрытой вкладке (на часах: запястье опущено).

## Картинка

Вся статика уровня запекается один раз в offscreen-canvas.

- **Дерево** светлое (клён/берёза), попиксельный генератор: value-noise fbm, волнистые волокна низкого контраста, без досок, швов и сучков. Пол светлее, стенки чуть темнее и теплее.
- **Свет уровня**: сверху лабиринта (в его локальных координатах). Круговой свет: подсветка центра уровня + затемнение к краям (source-atop поверх пола и стен).
- **Кромки стенок**: градиентные полосы 11 px в клипе пола — тёмная тень вдоль верхних кромок, светлая подсветка вдоль нижних, приглушённо-тёмная у вертикальных. На стенки не залезают.
- **Дырка**: колодец с радиальным градиентом + **одно** толстое кольцо 7 px со сплошным вертикальным градиентом (тёмное сверху → светлое снизу) + мягкая тень вокруг. Никаких двойных обводок.
- **Шарик — хром**: радиальная база (белый верх-слева → графит #343c45 к низу), тёмная размытая полоса посередине, тёмные смазанные отражения по бокам, тёплый отсвет дерева снизу, затемнение по ободу, один блик с гало сверху-слева. Мелких вторичных бликов нет.
- **Отражение уровня в шарике**: текстура уровня внутрь клипа шара, масштаб 0.35, поворот на θ, альфа 0.32 — между базой и хромовыми слоями. Подъехал к стенке — она видна в металле.
- **Тень шарика**: радиальная из центра во все стороны (0.4 у кромки, резкий фоллофф до 1.45r), рисуется в системе лабиринта с клипом по П-контуру — на стенки не попадает.

## Что уже пробовали и отвергли

- **Паркет из досок с сучками.** Слишком контрастный и пёстрый. Дерево — светлое, спокойное, без досок.
- **Резкий горизонт в хроме.** Шарик выглядел как глаз/шлем. Хром — светлый, с размытыми отражениями.
- **Тень-эллипс под шариком.** Торчала «грыжей» снизу. Тень радиальная из центра, вид сверху.
- **Двойная обводка дырки.** Читалась как унитазное сиденье. Одно кольцо с градиентом света.
- **Вращающаяся миникарта.** Дезориентирует. Статична всегда.
- **Трение о стены 5/с.** Убивало инерцию — шарик вставал как вкопанный. 1.1/с.
- **Мелкий вторичный блик на шарике.** И экранный, и вращающийся варианты мусорили. Убран совсем.
- **Шейдинг шарика, вращающийся с лабиринтом.** Ломает металл. Вращается только отражение уровня.

## Что проверять перед тем, как сказать «готово»

1. `npx tsc -b && npx vite build` зелёные.
2. Инерция: при постоянном наклоне скорость качения растёт, шарик докатывается по инерции.
3. Отпускание драга короны в любом месте страницы отвязывает колесо.
4. Измеритель никогда не показывает «-0.00».
5. Полный проход: старт → левая нога → перекладина → правая нога → дырка → «ФИНИШ» → тап → рестарт.
6. Тень шарика не залезает на стенки; стрелка гаснет у дырки; миникарта не вращается.
